C. Types

  • primitives(value) types
    number, string, boolean, symbol, null, undefined
    • copied by their values
  • reference/complex types
    Object, Function, Array
    • copied by their reference: the reference or address is copied, point to the same memory
    • Use const(block-scoped) for all of the references

D. Strings

1. Rules
  • Use single quotes ' ' for strings

  • When programmatically building up strings, use template strings instead of concatenation

  • no eval() on a string

E. Arrays

Rules
  • Use array spreads ... to copy arrays

  • Use array destructuring

  • Use Array.from for converting an array-like object to an array.

F. Object

1. create Object
  • object literal
  • facroty functions
  • constructor functions
    2. methods
3. special objects
  • Math
  • String
  • Date
Rules
  • Use the literal syntax for object creation, no new object

  • Use object method shorthand and property value shorthand

  • Group the shorthand properties at the beginning of the object

  • Only quote properties that are invalid identifiers

  • Do not call Object.prototype methods drectly, such as object.hasOwnProperty(key)

  • Prefer the object spread operator over Object.assign to shallow-copy objects. Use the object rest operator to get a new object with certain properties omitted.
