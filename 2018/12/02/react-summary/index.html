      4. JSX
  • JavaScript XML, just a notation
  • JSX expressions are just like normal JavaScript object. We can return them from a function or pass them to a function
  • Babel is a modern JavaScript compiler, convert JSX syntax to plain JavaScript code that the broswer can understand
  • JSX expressions must have one parent element
    • <div></div>
    • <React.Fragment></React.Fragment>
5. Name rules
  • the name of the file: camel notation
  • the name of the Component: pascal notation
    • this is how React knows to render a Component
  • the name of the event handler: camel notation
    keep the word from actual action
    • For props: prefix with on*, as in onClick
    • For function names: prefix with handle* as inhandleClick
    • For complicated naming: put the noun first, then the verb, as in onAlertClick={handleAlertClick} onFormSubmit={handleFormSubmit} onRegistrationSubmit onLoginSubmit

D. Composing Components

1. Passing data to Component
  • we need to use props to pass the data
  • passing children: if we wanna to pass a complex elements to a child component, we use the children prop
    2. Props vs State
  • props contains the data we give to a component, whereas state includes data that is local or private to that component
  • the state is local and internal to that component and invisible to other components
  • props is read only
3. Single source of truth
  • controlled component doesn’t have its own state. It receives data from props and whenever data needs to be changed, this component will raise an event
  • lift the state up: bubbling it up to the parent of this component
    • it allows to pass data between siblings
    • make re-usability easier
4. Stateless Functional Components
  • cannot use lifecycle hooks
5. Lifecycle Hooks


3 phases

  • Mount

    • constructor:

      • is called only once, when the instance of the class is created
      • a good place to set the state from the props from the outside, don’t forget the super(props) in this case
      • cannot call setState, only can be called when the component is rendered and place in the DOM

    • static getDerivedStateFromProps()

    • render

      • when one component is rendered, all of it’s children are also rendered recursively
    • componentDidMount
      • is called after the component is rendered into the DOM
      • perfect place to get AJAX data from the server
        • the data will not be there before render happens
        • with the new asynchronous rendering in React Fiber

  • UPDATE
    happened when the state or the props of the component changes
    entire componet tree is rendered, updating the virtual DOM

    • componentDidUpdate

  • UNMOUNT

    • componentWillUnmount
      • before unmount we can remove the setTimeout or some listenings
